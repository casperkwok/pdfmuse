# Releasing pdfmuse

One git tag (`vX.Y.Z`) drives a synchronized release across crates.io, PyPI, and
npm via [`.github/workflows/release.yml`](.github/workflows/release.yml).

## Package naming (decision A)

| Ecosystem | Package | Source crate | Notes |
|---|---|---|---|
| crates.io | `pdfmuse-core` (library) | `crates/pdfmuse-core` | `cargo add pdfmuse-core` |
| crates.io | `pdfmuse` (CLI, optional) | `crates/pdfmuse-cli` | `cargo install pdfmuse` → `pdfmuse` binary |
| PyPI | `pdfmuse` | `crates/pdfmuse-python` (maturin, abi3) | `pip install pdfmuse` |
| npm | `pdfmuse` *(or scoped — see below)* | `bindings/node` (napi) | native binding |
| npm | `@pdfmuse/core` | `crates/pdfmuse-wasm` (wasm-pack) | browser/WASM |

> The library is `pdfmuse-core`; the bare name `pdfmuse` is the CLI + umbrella.

> filter (vs `pdfmake`); appeal pending. If it's not granted, publish the native
> binding as `@pdfmuse/node` and update `bindings/node/package.json` + README.

## One-time setup

1. **GitHub repo secrets** (Settings → Secrets → Actions):
   - `CARGO_REGISTRY_TOKEN` — crates.io API token
   - `NPM_TOKEN` — npm automation token (org `@pdfmuse`)
   - `PYPI_API_TOKEN` — or, preferred, configure **PyPI Trusted Publishing (OIDC)**
     for this repo and drop the token from `release.yml`.
2. Ensure `pdfmuse-core` (and `pdfmuse` if publishing the CLI) name ownership on
   crates.io; `@pdfmuse` org + `pdfmuse` on PyPI already reserved.

## Pre-flight (local, no upload)

```bash
cargo test && cargo test --features rayon           # unit + snapshot + robustness
cargo clippy --workspace --all-targets -- -D warnings
python tests/parity/check.py                        # cross-binding byte-identical
cargo publish --dry-run -p pdfmuse-core             # crates.io packaging
maturin build --release --manifest-path crates/pdfmuse-python/Cargo.toml -o dist
wasm-pack build crates/pdfmuse-wasm --target web --release --out-dir pkg
```

## Cut a release

1. **Bump the version** — single source of truth is `[workspace.package] version`
   in the root `Cargo.toml`. Also update:
   - `bindings/node/package.json` `"version"` (napi main package)
   - (WASM `pkg/package.json` is generated by wasm-pack from the crate version)
2. Commit: `git commit -am "release: v0.1.0"`.
3. Tag and push:
   ```bash
   git tag v0.1.0
   git push origin main --tags
   ```
   `release.yml` runs `check-version` (tag must equal the workspace version),
   then publishes crates.io / PyPI wheels / builds Node / publishes WASM.

## Post-publish verification (clean env)

```bash
cargo add pdfmuse-core                               # (in a fresh crate)
pip install pdfmuse && python -c "import pdfmuse"
npm init -y && npm i @pdfmuse/core                   # (or pdfmuse once available)
```

## Rollback

- **crates.io**: `cargo yank --version X.Y.Z pdfmuse-core` (cannot delete; yank
  hides it from new resolves).
- **npm**: `npm deprecate pdfmuse@X.Y.Z "…"` (unpublish only within 72h).
- **PyPI**: delete the release in the web UI (the version number is then burned).
