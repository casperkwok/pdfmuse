This page covers how to use the [pdfmuse](https://github.com/casperkwok/pdfmuse) [document loader](/oss/integrations/document_loaders) to load PDF and DOCX files into LangChain.

`pdfmuse` is a deterministic PDF/DOCX parser aimed at RAG: the same file always produces the same output, it extracts text with exact page coordinates, tables and section structure, and it runs entirely locally with no ML dependencies in its core path. The `langchain-pdfmuse` package wraps it as a `BaseLoader`.

## Setup

### Credentials

No credentials are required. `PdfmuseLoader` parses files locally, and nothing is sent to a remote service.

### Installation

Install the `langchain-pdfmuse` package. It pulls in `pdfmuse` (a prebuilt wheel, no Rust toolchain needed):

```python
pip install -qU langchain-pdfmuse
```

## Initialization

Now we can instantiate our loader. `PdfmuseLoader` supports three modes:

- `"single"`: one `Document` for the whole file
- `"page"` (default): one `Document` per page
- `"elements"`: one `Document` per structural block (heading / paragraph / table), with section-aware metadata (`page`, `category`, `heading_path`, `bbox`)

```python
from langchain_pdfmuse import PdfmuseLoader

loader = PdfmuseLoader(
    "./example.pdf",
    mode="elements",
)
```

## Load

```python
docs = loader.load()
docs[0]
```

```output
Document(metadata={'source': './example.pdf', 'source_kind': 'Pdf', 'page': 0, 'category': 'Title', 'heading_path': ['Experience'], 'bbox': {'x0': 72.0, 'y0': 96.0, 'x1': 210.0, 'y1': 110.0}}, page_content='Experience')
```

```python
print(docs[0].metadata)
```

```output
{'source': './example.pdf', 'source_kind': 'Pdf', 'page': 0, 'category': 'Title', 'heading_path': ['Experience'], 'bbox': {'x0': 72.0, 'y0': 96.0, 'x1': 210.0, 'y1': 110.0}}
```

In `"elements"` mode each chunk carries where it came from: its page, whether it is a `Title` / `NarrativeText` / `Table`, its section breadcrumb (`heading_path`) and its bounding box (`bbox`) on the page. Because the parser is deterministic, re-running the loader on the same file yields identical Documents, so your index does not drift between runs.

## Lazy load

```python
pages = []
for doc in loader.lazy_load():
    pages.append(doc)
```
